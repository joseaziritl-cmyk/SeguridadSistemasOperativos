Gestión del Procesador y la Memoria: Protección de Memoria en Sistemas Multiproceso



Por Juan Zárraga | Octubre 24, 2025


En un sistema multiproceso, la protección de memoria se divide en dos categorías: la protección entre procesos y la protección del sistema operativo (kernel) frente a los procesos.

Protección entre Procesos

Los procesos utilizan direcciones virtuales, que son traducidas a direcciones físicas mediante una unidad de gestión de memoria (MMU). Cada proceso posee un espacio de direcciones virtuales único, lo que impide que un proceso acceda a la memoria de otro. Durante un cambio de contexto, se actualiza la tabla de páginas que mapea estas direcciones, garantizando así la seguridad y el aislamiento entre procesos.

Protección del Kernel

El kernel no opera como un proceso aislado; debe permitir que ciertos componentes sean ejecutados por los procesos, protegiendo al mismo tiempo su propio espacio de memoria. Para ello, se implementan niveles de privilegio de ejecución, donde existen dos niveles: el nivel de kernel y el nivel de usuario. El nivel de kernel permite ejecutar instrucciones privilegiadas y acceder a todos los registros, mientras que el acceso a áreas del kernel está restringido. Cuando un proceso intenta acceder a una zona de memoria, se verifica su nivel de privilegio; si no es suficiente, se lanza una excepción. El kernel tiene su propia tabla de traducción de direcciones, que permanece constante durante los cambios de contexto y se activa cuando el control pasa al kernel. El cambio al modo kernel se realiza a través de puntos de entrada específicos mediante llamadas al sistema.

Funciones Básicas de un Sistema Operativo

Las funcionalidades de un sistema operativo se pueden clasificar en cuatro grupos principales:

1. Gestión del Procesador y la Memoria: Asegura que cada proceso reciba tiempo del procesador y gestiona la asignación de memoria.

2. Gestión de Sistemas de Almacenamiento: Define cómo se almacenan los datos.

3. Gestión de Entrada/Salida: Administra la interacción entre hardware y aplicaciones.

4. Gestión de la Red: Facilita la comunicación en red entre aplicaciones, destacándose como una función crítica.
